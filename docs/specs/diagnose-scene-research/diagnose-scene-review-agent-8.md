# Agent 8 — SOFA component reference docs

**Source:** `https://sofa-framework.github.io/doc/components/...` (verbatim quotes flagged Y; absent fields flagged N).
**Scope:** the 15 classes listed in `2026-04-25-diagnose-scene-design.md` API section.

## Executive summary

Every component in the spec's `printLog` target list exposes `printLog` as a real Data field and the docs all describe it identically as *"if true, emits extra messages at runtime"* — but the docs do **not** specify per-component what that message stream actually contains, so verifying useful output requires runtime testing, not docs. **`SparseDirectSolver` does not exist as a component name in SOFA**: the project either means `SparseLDLSolver`, `SparseLUSolver`, `SparseCholeskySolver`, `AsyncSparseLDLSolver`, `SVDLinearSolver`, or `BTDLinearSolver` — the spec must drop it or replace it. **Agent 1's M4 is confirmed**: `UncoupledConstraintCorrection.verbose` and `LCPConstraintSolver.displayDebug` are separate gates from `printLog`, so a class-name + `printLog` walk silently misses output for those two. **Agent 1's M5 is confirmed and refined**: `NNCGConstraintSolver` and `BlockGaussSeidelConstraintSolver` expose `currentIterations`, `currentError`, `currentNumConstraints` as **OUTPUT Data fields** (queryable via `findData`), so `solver_iter_cap_hit` for those two does *not* require log parsing — but `LCPConstraintSolver` exposes neither, so for it the rule still needs regex on logs. `EulerImplicitSolver` additionally exposes a `residual` Data field (default 1.79769e+308 ≈ DBL_MAX) and a `computeResidual` toggle, which would make a strong v2 smell test for "Newton solve diverging."

## Recommendations to revise the spec's `printLog` target list

1. **Drop `SparseDirectSolver`.** It is not a SOFA component. Keep `SparseLDLSolver`, and consider adding `SparseLUSolver`, `SparseCholeskySolver`, `AsyncSparseLDLSolver`, `SVDLinearSolver`, `BTDLinearSolver`.
2. **`QPInverseProblemSolver`** is already in spec §2 step 2 but missing from the `validate_scene` list in `CLAUDE.md` (separate concern from `diagnose_scene`).
3. **Implement Agent 1 M4 fix** as documented: also flip `verbose` (UncoupledConstraintCorrection) and `displayDebug` (LCPConstraintSolver). Use `obj.findData("verbose")` truthy → `setValue(True)`, same for `displayDebug`. **Required**, not optional — the bare `printLog` walk silently misses output for two of the priority components.
4. **Refine Agent 1 M5**: `solver_iter_cap_hit` can read `currentIterations` directly via `findData` for `NNCGConstraintSolver` and `BlockGaussSeidelConstraintSolver`. Only `LCPConstraintSolver` (and `CGLinearSolver`) require log parsing. The spec should branch on solver class.
5. **Add a smell test using `EulerImplicitSolver.residual`**: when `computeResidual=True`, post-step `residual` Data field is queryable. A non-decreasing residual or residual > 1.0 is a strong "Newton not converging" signal — much more reliable than log scraping.
6. **Add a smell test for `DefaultAnimationLoop` + Lagrangian constraint solver pairing**: docs explicitly state this combination is unsupported. Walk: if root has `DefaultAnimationLoop` AND any descendant has `NNCGConstraintSolver|LCPConstraintSolver|BlockGaussSeidelConstraintSolver|GenericConstraintSolver`, emit `animation_loop_constraint_mismatch`.
7. **Add a smell test for missing `constraintSolver` link on `FreeMotionAnimationLoop`**: silent fallback to auto-created LCP masks bugs. Read the link, warn if it auto-resolved.
8. **Add `wire_optimization` discoverability** to the playbook: for cable/tendon scenes (priority for this project's soft-robotics domain), `LinearSolverConstraintCorrection.wire_optimization=1` is documented and likely under-used.

## Contradictions / additions vs. Agent 1 and Agent 4

- **Confirms Agent 1 M4** (component-specific gates): `UncoupledConstraintCorrection.verbose` and `LCPConstraintSolver.displayDebug` are real, documented, distinct from `printLog`. Spec must mutate both.
- **Refines Agent 1 M5**: not all constraint solvers need log parsing. NNCG and BlockGaussSeidel expose `currentIterations` as output Data. LCP does not. Branch on class.
- **Does not confirm Agent 4 pitfall #4** (`UncoupledConstraintCorrection` + deformable FEM): the component reference page **does not** document this incompatibility. It is community/forum knowledge. Tag in the playbook as `[community]` rather than as documented.
- **Confirms a new "documented" pitfall not yet in the spec or Agent 4 list**: `DefaultAnimationLoop` does not support Lagrange-multiplier constraints. Easy class-pair check, high diagnostic value.
- **Spec bug**: `SparseDirectSolver` is not a SOFA component. The name appears in `CLAUDE.md` validation list and in the diagnose_scene spec — both should be corrected. Verify also in `sofa_mcp/architect/scene_writer.py`.
