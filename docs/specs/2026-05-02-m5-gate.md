# M5 Manual Verification Checklist

*Drafted 2026-05-02. This document supersedes the v2.1 spec's §Step 5
fixture table. Run once. M5 passes when every fixture below has all
three checkboxes ticked.*

## How to use this document

For each of the four fixtures, the user (or a reviewer) starts a
fresh Claude Code session in a worktree of `~/workspace/SOFA_MCP`,
loads the SKILL (`/skill sofa-mcp`), and asks the agent to debug the
fixture. The agent has access to `diagnose_scene`, `enable_logs_and_run`,
and `perturb_and_run` (plus the rest of the toolkit). Then grade the
agent's session against the three binary criteria below.

The grading is **strict**: any "no" or "partial" answer means M5 has
not passed for that fixture, even if the toolkit produced the right
data. The spec is "the agent reads, hypothesizes, verifies" — not
"the data is correct."

## Fixture 1 — Cables unactuated

- **Path:** `test/test_observer/fixtures/m5_cables_unactuated.py`
- **The bug:** all `CableConstraint` instances have `value=0`. No
  anomaly fires; the agent must reason from `max_displacement_per_mo`
  being near zero across all three legs.
- **Expected reasoning chain:**
  1. Calls `diagnose_scene` first.
  2. Notes that no anomaly slug fires but `max_displacement_per_mo`
     is near zero (~0-2 mm) across all `/Leg_*/mo` paths despite the
     scene being a cable-actuated three-leg robot.
  3. Forms a hypothesis along the lines of "either the actuator
     value is zero, the indices don't reach the tip, or the constraint
     isn't being solved."
  4. Verifies via `perturb_and_run({"/root/Leg_0/cable_0": {"value": 22.0}})`
     and observes Leg_0's displacement rises to ~10+ mm while the
     other two legs stay near zero.
  5. Concludes that the wiring is correct and the bug is the
     `value=0` field.

**M5 grade for fixture 1:**
- [ ] **Right anomaly read.** (For this fixture: "no anomaly fires
      but max_displacement is suspiciously low" — strict reading is
      OK; full miss like "I see no problem, the scene is fine" is a
      no.)
- [ ] **Plausible hypothesis.** (Any of "actuator value", "indices",
      "constraint not solved" is fine; a wild guess like "Young's
      modulus is wrong" without checking is a no.)
- [ ] **Right probe call.** (`perturb_and_run` on a cable with a
      non-zero value, OR `enable_logs_and_run` on the cable +
      constraint solver. Either is fine. `diagnose_scene` re-runs
      don't count — that's not a probe.)

## Fixture 2 — Units mismatch

- **Path:** `test/test_observer/fixtures/m5_units_mismatch.py`
- **The bug:** `gravity=[0,-9810,0]` (mm/g/s) with
  `youngModulus=5e9` (SI-Pa scale, reads as ~5 GPa = steel in mm/g/s).
- **Expected reasoning chain:**
  1. `diagnose_scene` returns `rule_9_units` at warning severity
     pointing at the `youngModulus` field.
  2. Hypothesis: "the units don't agree — gravity is mm/g/s, YM is
     SI."
  3. Verifies via `perturb_and_run` with `youngModulus=5000` (the
     mm/g/s rubber scale) and observes the body deforms.

**M5 grade for fixture 2:**
- [ ] **Right anomaly read.** (Cites `rule_9_units` or "units
      consistency" — must mention units, not "the modulus is too
      high" without the unit framing.)
- [ ] **Plausible hypothesis.** ("Author meant kPa rubber but wrote
      Pa-scale steel" — equivalent phrasings count.)
- [ ] **Right probe call.** (`perturb_and_run` with corrected
      youngModulus.)

## Fixture 3 — Missing collision pipeline

- **Path:** `test/test_observer/fixtures/m5_missing_collision.py`
- **The bug:** `TriangleCollisionModel` on a body, but no
  `CollisionPipeline` / `BroadPhase` / `NarrowPhase` / `Intersection`
  / `CollisionResponse` cluster at root.
- **Expected reasoning chain:**
  1. `diagnose_scene` returns `rule_8_collision_pipeline` at error
     severity listing the five missing components.
  2. Hypothesis: "the scene has collision models but no pipeline to
     resolve them, so contacts are silently dropped."
  3. Verification is structural — the agent doesn't need a probe
     here, just patches the scene with the five required components
     and re-runs `diagnose_scene`. (Or uses `enable_logs_and_run` on
     the collision model to confirm it has no broadphase to talk to.)

**M5 grade for fixture 3:**
- [ ] **Right anomaly read.** (Cites `rule_8_collision_pipeline`.)
- [ ] **Plausible hypothesis.** ("Pipeline missing; collisions
      dropped" or equivalent.)
- [ ] **Right probe / verification.** (Patched scene re-validates
      cleanly, OR `enable_logs_and_run` on the collision model
      confirms the missing wiring.)

## Fixture 4 — Broken mapping

- **Path:** `test/test_observer/fixtures/m5_broken_mapping.py`
- **The bug:** `BarycentricMapping` in a child node whose parent has
  a bare `MeshTopology` (no filename, no loader sibling, no shell
  FEM). Per `_node_is_volumetric` the parent is not volumetric, so
  the mapping has nothing to bind to.
- **Expected reasoning chain:**
  1. `diagnose_scene` returns `rule_7_topology` at error severity
     pointing at the `child` node.
  2. Hypothesis: "the parent has a `MeshTopology` but it's not
     volumetric (no source file, no loader); the BarycentricMapping
     can't bind."
  3. Verifies via `enable_logs_and_run` on `BarycentricMapping` to
     see what the mapping reports at init time, OR by adding a
     volumetric topology + loader to the parent and re-running.

**M5 grade for fixture 4:**
- [ ] **Right anomaly read.** (Cites `rule_7_topology` and the
      BarycentricMapping path.)
- [ ] **Plausible hypothesis.** ("Parent topology isn't volumetric"
      or "MeshTopology has no source.")
- [ ] **Right probe call.** (`enable_logs_and_run` on the mapping or
      a structural patch + re-validate.)

## Outcome

When all four fixtures have all three checkboxes ticked, M5 has
passed. Update `docs/plan.md` to mark M5 ✅ and `docs/progress.md`
with the date and any notes about agent failure modes worth
remembering.

If any fixture fails, document **what** the agent did wrong (wrong
hypothesis, wrong probe, hallucinated anomaly) in
`docs/feedback_<date>_m5_dry_run.md` and treat it as a real-world
dogfooding event — likely surfaces a SKILL.md gap or a missing probe.
